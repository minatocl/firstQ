# 受付スキャナ → ダイナミクス自動受付 セットアップ引き継ぎ

> このファイルは **受付PC(Windows・ダイナミクスとスキャナが接続された端末)の Claude** 向けの引き継ぎメモです。
> Mac 側で作った「デジタル診察券(Apple/Google Wallet)」の続きで、**受付側の自動化**を担当します。

## ゴール
患者が Wallet の診察券QRを受付の据置スキャナにかざすと、**ダイナミクス(レセコン一体型電子カルテ)の受付一覧に自動で載る**ようにする。

## 前提・既知の事実
- **QRの中身 = カルテ番号(数字のみ)**。URLや接頭辞は一切含まない(受付スキャナがそのまま打鍵できるよう設計済み)。
- **スキャナ:BUSICOM BC-NL3000U II**(USB接続・**HIDキーボードエミュレーション**)。読み取ると「数字＋(設定した)サフィックス」を、いまカーソルがある欄にキー入力する。
  - 工場出荷に戻す設定バーコード:`@SETUPE1`(設定開始)→ `@FACDEF`(初期化)→ `@SETUPE0`(設定終了)。
  - **プレフィックス/サフィックス(Enter/Tab/Fキー付加)の設定バーコードは、付属クイックガイドには無い**。BUSICOM 公式の「BC-NL3000U II 設定マニュアル(PDF)」に載っている → busicom.co.jp からダウンロードして参照。
- **対象システム:ダイナミクス(Dynamics / superdyn.jp)** = 日本のクリニック用 Windows デスクトップ電子カルテ。**バーコード診察券は標準機能**(カルテ番号をバーコードで扱う運用が公式にある)。
- 環境:Windows の受付PC。医療本番システムなので**診療時間外に検証**すること。

## 進め方(シンプル順・上から試す)
### ① まず素の挙動を確認(ソフト不要が理想)
ダイナミクスの**受付画面のカルテ番号入力欄**にカーソルを置いた状態でスキャン。
- それだけで受付一覧に載る/カルテが開くなら → **スキャナに Enter サフィックスを設定するだけで完了**。
- 「受付」画面の入力欄・確定キー(Enter か 別のキーか)を実機で確認する。ダイナミクスのマニュアル/ヘルプ、または販売店・[ユーザーメーリングリストQ&A](http://www7a.biglobe.ne.jp/~dynamics/dynaml.htm) に受付バーコード運用の記載があるはず。

### ② スキャナのプレフィックス/サフィックスで完結
①で「追加ボタン→欄フォーカス」等の前操作が要る場合、スキャナに
`[新規受付を開くキー(Fキー等)] + [番号] + [Enter]`
を送らせる。ダイナミクス側に新規受付のショートカットがあればこれで一発。設定は BUSICOM 設定マニュアルの該当バーコードを読ませる。

### ③ AutoHotkey 常駐スクリプト(= カーソル操作なしの完全自動)
①②で足りない/「カルテ番号欄にカーソル」工程も無くしたい場合。受付PCに軽い AutoHotkey スクリプトを常駐させ、**スキャンを横取りして自動誘導**する:
- スキャナ出力に**目印プレフィックス**(先頭に特定文字。マニュアル165-166)＋Enterサフィックス(p199)を設定。
- AHK がプレフィックスを検知したら以降の数字を Enter まで取り込み、**元の入力は握りつぶす**(別ウィンドウへの誤入力を防止=安全)。
- そのうえで「Dynamics受付ウィンドウを前面化(WinActivate)→ カルテ番号欄をフォーカス(Dynamicsのショートカット、無ければ受付画面の固定座標をクリック)→ 番号入力 → Enter確定」を自動実行。
- これで**スタッフのフォーカス操作ゼロ**。患者がかざすだけで受付一覧に載る。
- **この受付PCの Claude で、実際のダイナミクス画面を見ながら作成・調整する。**
- スキャナがファンクション/制御キーをプレフィックスで送れることは確認済み(マニュアル p59-61)。

### ④ 安全策(医療なので必須)
誤スキャン→誤受付を防ぐため、**読み取った番号の患者氏名を一瞬表示して目視確認**する一手間を推奨。完全ノールック全自動は避ける。

## Mac 側の関連情報(参考)
- 診察券発行/照合 Worker:`minato-card`(Cloudflare)。QR = カルテ番号(数字5桁ちょうど)。
- リポジトリ:`minatocl/firstQ`(このファイルもそこにある)。card/index.html = 患者の照合ページ、admin.html = 受付の発行パネル。
- Apple Wallet 本番稼働、Google Wallet は公開審査待ち(承認後に全Android患者可)。
